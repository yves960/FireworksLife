## 为什么换？

最近把博客从复杂的全栈架构迁移到了纯静态站点，体验非常棒。

之前的 FireworksLife 有完整的后端（FastAPI）和数据库（SQLite），功能很全但也很重：
- 需要维护服务器
- 每次部署要跑 Docker
- 其实大部分功能根本用不上

我只是想写写文章，又不是做个社交平台……

## 技术选型

### Astro

[Astro](https://astro.build) 是专门为内容型网站设计的框架，核心理念是 **"Ship less JavaScript"**：

- 默认零 JS 输出
- 支持 React/Vue/Svelte 组件按需使用
- 内置 Markdown/MDX 支持
- 构建速度极快

### Netlify

[Netlify](https://netlify.com) 是静态站点的最佳托管平台：

- **免费额度够用** — 100GB 带宽/月
- **全球 CDN** — 你的网站从离用户最近的节点分发
- **自动 HTTPS** — 免费 SSL 证书
- **一键部署** — GitHub 推送代码自动部署

## 搭建过程

整个过程不到 30 分钟：

1. 在 GitHub 创建仓库
2. 本地创建 Astro 项目
3. 推送到 GitHub
4. Netlify 导入 GitHub 仓库
5. 完成！🎉

## 炫酷效果

为了不让博客太无聊，加了一些视觉特效：

- 🌟 **粒子背景** — 星星点点的动效
- ✨ **渐变标题** — 紫色渐变呼吸动画
- 💫 **卡片悬停** — 发光边框效果
- 🎯 **流畅过渡** — 页面切换动画

所有动画都是 CSS 实现，没有引入额外的 JS 库，保持了极高的性能。

## 总结

这次重构最大的收获是：**有时候少就是多**。

去掉不需要的功能之后，网站变得更快、更容易维护、也更纯粹了。

如果你也想搭一个个人博客，强烈推荐试试 Astro + Netlify 这个组合！
