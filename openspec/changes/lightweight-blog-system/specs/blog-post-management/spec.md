## ADDED Requirements

### Requirement: 创建博客文章
系统必须允许已认证用户创建新的博客文章，包含标题、Markdown内容、标签和状态。

#### Scenario: 成功创建文章
- **当** 管理员在后台提交新文章表单
- **那么** 系统创建文章记录并保存到数据库
- **并且** 文章ID自动生成
- **并且** 创建时间自动记录
- **并且** 响应状态码为201

#### Scenario: 创建草稿文章
- **当** 管理员选择状态为"草稿"并提交
- **那么** 文章以草稿状态保存
- **并且** 文章不会在前台显示

### Requirement: 阅读博客文章
系统必须允许用户通过文章ID或slug阅读已发布的博客文章。

#### Scenario: 通过ID读取文章
- **当** 用户访问文章详情页面并提供有效的文章ID
- **那么** 系统返回完整的文章信息
- **并且** Markdown内容被渲染为HTML
- **并且** 响应状态码为200

#### Scenario: 读取不存在的文章
- **当** 用户访问不存在的文章ID
- **那么** 系统返回404状态码
- **并且** 显示"文章不存在"错误信息

### Requirement: 更新博客文章
系统必须允许已认证用户更新现有的博客文章内容。

#### Scenario: 更新文章标题
- **当** 管理员修改文章标题并保存
- **那么** 文章标题被更新
- **并且** 更新时间自动记录
- **并且** 响应状态码为200

#### Scenario: 发布草稿文章
- **当** 管理员将草稿状态更改为"已发布"
- **那么** 文章状态变为已发布
- **并且** 文章在前台可见

### Requirement: 删除博客文章
系统必须允许已认证用户删除博客文章。

#### Scenario: 删除文章
- **当** 管理员确认删除操作
- **那么** 文章从数据库中移除
- **并且** 响应状态码为200
- **并且** 返回删除成功的确认消息

#### Scenario: 删除不存在的文章
- **当** 管理员尝试删除不存在的文章
- **那么** 系统返回404状态码

### Requirement: 文章列表查询
系统必须支持分页查询已发布的博客文章列表。

#### Scenario: 获取第一页文章列表
- **当** 用户请求文章列表且不提供页码
- **那么** 系统返回第一页（10篇文章）
- **并且** 包含文章总数和总页数
- **并且** 按创建时间降序排列

#### Scenario: 按标签筛选文章
- **当** 用户提供标签参数查询文章
- **那么** 系统返回包含该标签的文章
- **并且** 响应状态码为200
