## ADDED Requirements

### Requirement: 用户名密码登录
系统必须支持用户名密码登录验证。

#### Scenario: 正确的用户名密码登录
- **当** 用户提交正确的用户名和密码
- **那么** 系统验证凭据成功
- **并且** 生成JWT Token
- **并且** Token通过HTTP-Only Cookie返回
- **并且** 响应状态码为200

#### Scenario: 错误的用户名密码登录
- **当** 用户提交错误的用户名或密码
- **那么** 系统拒绝登录请求
- **并且** 返回401状态码
- **并且** 显示"用户名或密码错误"提示

### Requirement: 邮箱密码登录
系统必须支持使用邮箱作为登录凭证。

#### Scenario: 正确的邮箱密码登录
- **当** 用户提交正确的邮箱和密码
- **那么** 系统验证凭据成功
- **并且** 生成JWT Token
- **并且** Token通过HTTP-Only Cookie返回
- **并且** 响应状态码为200

#### Scenario: 邮箱不存在
- **当** 用户提交不存在的邮箱
- **那么** 系统拒绝登录请求
- **并且** 返回401状态码
- **并且** 显示"邮箱或密码错误"提示

#### Scenario: 邮箱格式验证
- **当** 用户提交无效的邮箱格式
- **那么** 系统显示"邮箱格式不正确"错误提示
- **并且** 响应状态码为400

### Requirement: 邮箱验证和绑定
系统必须支持邮箱验证和绑定功能。

#### Scenario: 发送验证邮件
- **当** 用户在设置中绑定新邮箱
- **那么** 系统发送包含验证码的邮件
- **并且** 验证码有效期30分钟
- **并且** 显示"验证邮件已发送"提示

#### Scenario: 验证邮箱验证码
- **当** 用户提交正确的邮箱验证码
- **那么** 系统验证邮箱成功
- **并且** 更新用户邮箱状态为已验证
- **并且** 显示"邮箱验证成功"提示

#### Scenario: 验证码过期
- **当** 用户提交过期的验证码
- **那么** 系统拒绝验证
- **并且** 显示"验证码已过期，请重新获取"提示

### Requirement: 微信OAuth登录
系统必须支持微信OAuth2.0授权登录。

#### Scenario: 微信扫码登录
- **当** 用户点击"微信登录"按钮
- **那么** 系统重定向到微信授权页面
- **并且** 显示二维码供用户扫码
- **并且** 授权回调地址配置为系统后台

#### Scenario: 微信授权成功
- **当** 用户在微信端确认授权
- **那么** 微信重定向回系统并携带code参数
- **并且** 系统使用code换取access_token和openid
- **并且** 获取用户基本信息（昵称、头像）
- **并且** 创建或更新用户记录
- **并且** 生成JWT Token
- **并且** 响应状态码为200

#### Scenario: 微信授权失败
- **当** 用户拒绝微信授权
- **或者** 微信返回错误
- **那么** 系统显示"授权失败"提示
- **并且** 重定向回登录页面

#### Scenario: 微信用户首次登录
- **当** 微信用户首次通过OAuth登录
- **那么** 系统创建新用户记录
- **并且** 绑定微信openid
- **并且** 使用微信昵称作为默认用户名
- **并且** 保存微信头像URL

#### Scenario: 微信用户再次登录
- **当** 已绑定的微信用户再次登录
- **那么** 系统通过openid查找用户
- **并且** 更新用户微信信息
- **并且** 返回JWT Token

### Requirement: 用户登出
系统必须支持用户登出并使Token失效。

#### Scenario: 正常登出
- **当** 用户点击登出按钮
- **那么** 系统清除客户端的Token Cookie
- **并且** 清除微信登录态
- **并且** 返回登出成功确认
- **并且** 响应状态码为200

### Requirement: Token验证
系统必须验证每个需要认证的API请求中的Token有效性。

#### Scenario: 有效Token访问受保护资源
- **当** 请求包含有效的JWT Token
- **那么** 系统允许访问受保护资源
- **并且** 响应状态码为200

#### Scenario: 无效或过期Token
- **当** 请求包含无效或过期的Token
- **那么** 系统拒绝访问
- **并且** 返回401状态码
- **并且** 提示用户重新登录

#### Scenario: 缺少Token
- **当** 请求未包含Token且访问受保护资源
- **那么** 系统返回401状态码
- **并且** 提示需要登录

### Requirement: 密码安全存储
系统必须使用安全的哈希算法存储用户密码。

#### Scenario: 新用户注册
- **当** 创建新用户时
- **那么** 密码使用bcrypt算法哈希
- **并且** 明文密码不存储在数据库中
- **并且** 哈希包含随机盐值

### Requirement: 微信登录安全配置
系统必须正确配置微信OAuth安全参数。

#### Scenario: 配置微信AppID和AppSecret
- **当** 系统初始化时
- **那么** 从环境变量读取微信AppID和AppSecret
- **并且** 不得硬编码在代码中
- **并且** AppSecret仅用于服务端

#### Scenario: OAuth状态参数
- **当** 发起微信授权请求
- **那么** 包含随机生成的state参数
- **并且** state保存在会话中
- **并且** 回调时验证state防止CSRF攻击

### Requirement: 登录方式切换
系统必须支持多种登录方式的切换。

#### Scenario: 切换到微信登录
- **当** 用户在登录页点击"微信登录"
- **那么** 显示微信扫码界面
- **并且** 隐藏传统登录表单
- **并且** 添加平滑的过渡动画

#### Scenario: 返回传统登录
- **当** 用户在微信登录页点击"账号密码登录"
- **那么** 显示登录表单
- **并且** 隐藏微信扫码界面
- **并且** 添加平滑的过渡动画

### Requirement: 记住登录状态
系统必须支持"记住我"功能。

#### Scenario: 启用记住登录
- **当** 用户勾选"记住我"并登录
- **那么** JWT Token有效期设置为30天
- **并且** Cookie设置为持久化存储
- **并且** 下次自动登录

#### Scenario: 不记住登录
- **当** 用户未勾选"记住我"并登录
- **那么** JWT Token有效期设置为24小时
- **并且** Cookie设置为会话Cookie
- **并且** 关闭浏览器后需重新登录
